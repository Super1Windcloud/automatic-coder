# Interview Coder 用户使用教程

本文档面向最终用户，说明软件的安装、激活、配置和日常使用方法。建议随安装包一并分发。

## 1. 软件简介

Interview Coder 是一款桌面辅助工具，适用于笔试题、截图题、选择题和算法题场景。软件可以根据你截取的题目图片，调用视觉大模型生成答案、解析和代码。

主要能力：

- 截图识别题目内容
- 自动生成选择题答案和简要解析
- 自动生成算法题思路与代码
- 支持切换编程语言
- 支持切换截图区域
- 支持调整窗口透明度
- 支持在线更新

## 2. 安装说明

1. 双击安装包，按提示完成安装。
2. 安装完成后，启动软件。
3. 首次启动时，如果软件尚未激活，会自动弹出激活窗口。

说明：

- Windows 下建议允许软件正常访问网络，否则可能无法激活或无法调用模型。
- 如果安全软件弹出提示，请确认该程序来自可信发行方后再放行。

## 3. 首次使用流程

首次使用一般按下面顺序完成：

1. 启动软件
2. 复制机器码并申请许可证
3. 配置偏好设置
4. 截图题目
5. 查看答案

## 4. 激活软件

首次启动时会出现“输入许可证”窗口。

操作步骤：

1. 在激活窗口查看并复制当前机器码。
2. 将机器码发送给发行方或宿主机管理端。
3. 等待发行方签发该机器专属的许可证。
4. 将收到的完整许可证文本粘贴到激活窗口输入框中。
5. 点击“立即激活”。
6. 激活成功后，窗口会自动关闭并进入主程序。

常见提示说明：

- `激活成功，正在启动应用…`：激活已完成，可以开始使用。
- `许可证签名无效，请确认内容来源。`：许可证不是由发行方签发，或内容已被修改。
- `许可证格式错误，请重新粘贴完整内容。`：粘贴内容不完整，或不是有效许可证。
- `该许可证不属于当前机器，请重新签发。`：这份许可证绑定的是另一台机器。
- `该许可证已过期，请联系发行方续期。`：许可证已超过有效期。
- `该机器的许可证已被远程吊销。`：发行方已将当前机器从授权名单中移除。
- `暂时无法校验远程吊销列表，请联网后重试。`：当前无法完成联网校验。
- `激活系统尚未就绪，请稍后重试。`：请稍后再次尝试。
- `当前版本未启用激活校验。`：当前版本不需要激活。

注意事项：

- 许可证是按机器签发的，换机器后通常需要重新申请。
- 建议直接复制粘贴完整许可证，不要手动修改内容。
- 发行方可以远程吊销许可证。被吊销后，软件会在联网校验后自动失效。

## 5. 偏好设置说明

首次启动或需要修改配置时，会弹出“请选择偏好设置”窗口。之后也可以通过快捷键 `Alt+3` 或托盘菜单再次打开。

可配置项目如下。

### 5.1 编程语言

用于指定算法题默认输出的代码语言。当前可选项包括：

- TypeScript
- JavaScript
- Python
- Java
- Rust
- Golang
- Ruby
- C#
- C++
- C
- PHP
- Kotlin
- Swift
- Dart
- Scala
- Elixir

建议按你的笔试要求选择对应语言。

### 5.2 截图方位选择

用于指定快捷截图时优先截取哪个区域，可选：

- 屏幕左半边
- 屏幕右半边
- 屏幕上半边
- 屏幕下半边
- 全屏

如果你的题目通常出现在固定位置，建议预先设置好对应区域，截图更快。

### 5.3 提示词

提示词用于约束模型的回答方式。留空时，软件会使用默认提示词。

默认逻辑：

- 如果识别到是选择题，直接输出答案和简要解析
- 如果识别到是算法题，按你选择的编程语言输出代码和解题思路

如果你有更细的要求，也可以自行修改提示词，例如要求：

- 只输出最终答案
- 先给思路再给代码
- 代码添加注释
- 回答尽量简洁

### 5.4 大模型

软件支持在配置窗口中选择不同模型。不同模型在速度、稳定性和效果上可能有差异。通常保持默认即可。

### 5.5 API Key

配置窗口中提供了 `硅基流动 API Key` 输入框。只有在你的发行版本需要用户自己配置模型密钥时，才需要填写。

如果发行方已经内置或统一提供服务，可以忽略这一项。

填写建议：

- 只粘贴完整密钥，不要附带空格
- 保存后即可生效

### 5.6 确认保存

所有项目设置完成后，点击“确认选择”保存配置。

## 6. 主界面说明

主界面顶部会显示常用快捷键说明：

- `截图 (Alt+1)`
- `答案 (Alt+2)`
- `移动 (Alt+方向键)`
- `重置 (Alt+\`)`
- `隐藏 (Ctrl+Shift+\`)`

在已经截图后，主界面还会显示：

- 当前截图缩略图
- 当前编程语言
- 当前截图方位
- 当前提示词
- 当前大模型
- 当前页面透明度

## 7. 日常使用方法

### 7.1 基本操作流程

1. 将题目放到你屏幕上方便截取的位置。
2. 按 `Alt+1` 截图。
3. 按 `Alt+2` 生成答案。
4. 在主窗口中查看模型返回的结果。

建议在正式使用前先练习一次完整流程，确认截图区域和语言设置正确。

### 7.2 截图

快捷键：`Alt+1`

作用：

- 按当前设置的截图方位进行截图
- 如果已经存在旧截图，再次按下会先清空旧结果，再重新截图

建议：

- 确保题目内容完整、清晰，不要被遮挡
- 尽量只保留题干、选项、样例和输入输出要求，减少无关内容

### 7.3 生成答案

快捷键：`Alt+2`

作用：

- 对当前截图内容进行识别
- 调用模型生成答案
- 结果会展示在主窗口中

通常输出内容包括：

- 选择题答案
- 题目解析
- 算法题思路
- 对应编程语言代码

### 7.4 移动窗口

快捷键：

- `Alt+Up`
- `Alt+Down`
- `Alt+Left`
- `Alt+Right`

作用：

- 手动移动主窗口位置，避免遮挡当前屏幕内容

### 7.5 重置窗口

快捷键：`Alt+\``

作用：

- 重置窗口状态
- 清理当前显示位置偏移

如果你频繁移动过窗口，建议在位置错乱时使用此功能。

### 7.6 隐藏或显示窗口

快捷键：`Ctrl+Shift+\``

作用：

- 显示或隐藏主窗口
- 如果设置窗口处于打开状态，也会一起联动显示或隐藏

适合在需要快速收起界面时使用。

### 7.7 打开偏好设置

快捷键：`Alt+3`

作用：

- 重新打开配置窗口
- 修改语言、截图区域、提示词、模型等设置

### 7.8 切换模型

快捷键：`Alt+4`

作用：

- 快速切换当前使用的大模型

如果你发现当前模型效果一般，可以尝试切换后再重新截图和生成答案。

### 7.9 退出软件

快捷键：`Alt+5`

也可以通过系统托盘菜单选择“退出”。

## 8. 托盘菜单说明

软件运行后通常会驻留在系统托盘中。你可以通过托盘图标进行操作。

常用菜单项：

- `关于`
- `偏好设置`
- `切换模型`
- `页面透明度`
- `退出`

左键点击托盘图标时，主窗口会重新显示到前台。

## 9. 页面透明度设置

你可以通过托盘菜单中的“页面透明度”调整窗口透明度。

可选透明度：

- 100%
- 90%
- 80%
- 70%
- 60%
- 50%
- 40%
- 30%
- 20%

建议：

- 想看得更清楚时，使用 80% 到 100%
- 想降低存在感时，使用 20% 到 50%

透明度修改后会自动保存，下次启动仍会保留。

## 10. 软件更新

软件支持自动检查更新。

更新方式：

- 启动时自动检查新版本
- 快捷键 `Ctrl+F11` 可手动检查更新

当检测到新版本时，会弹出“应用更新”窗口。按提示完成安装即可。

建议：

- 保持网络畅通
- 更新前关闭不必要的程序
- 更新后重新启动软件

## 11. 常见问题

### 11.1 许可证无法通过

请依次检查：

- 是否复制完整许可证
- 是否多了空格或遗漏内容
- 当前许可证是否绑定当前机器
- 当前许可证是否已经过期
- 当前网络是否正常

如果仍然失败，请联系发行方重新签发许可证。

### 11.2 截图后没有结果

请检查：

- 是否已经先按 `Alt+1` 完成截图
- 截图区域是否正确
- 题目是否清晰可见
- 模型服务是否可用
- 是否已正确配置 API Key

### 11.3 答案不理想

建议尝试：

- 更换截图区域
- 截取更清晰、更完整的题目
- 修改提示词
- 切换模型
- 更换编程语言后重试

### 11.4 看不到主窗口

可以尝试以下方法：

- 按 `Ctrl+Shift+\`` 显示窗口
- 点击托盘图标
- 按 `Alt+\`` 重置窗口

### 11.5 配置窗口被关闭了

重新打开方式：

- 按 `Alt+3`
- 或通过托盘菜单进入“偏好设置”

## 12. 使用建议

- 首次使用先完成一次完整测试，确认激活、截图、出答案都正常。
- 尽量将题目放在固定区域，方便使用半屏截图。
- 如果是算法题，优先确认输出语言设置是否正确。
- 如果是选择题，截图时尽量包含所有选项和题干。
- 如果结果异常，先重新截图，再重新生成答案。

## 13. 联系支持

如果你在使用中遇到无法解决的问题，请联系软件发行方，并尽量同时提供以下信息：

- 软件版本号
- 问题出现时的操作步骤
- 报错提示截图
- 当前系统版本

如果发行方提供了专门客服或售后渠道，请以对方提供的联系方式为准。
